# Category 1: Data Integrity Audit

**Audited**: 2026-04-01
**Status**: IN PROGRESS

## Summary

The #1 problem confirmed: **pervasive LLM fallback generates fake data** when Apollo is unavailable or returns no results.

---

## Item-by-item audit

### 1.1 Company enrichment uses REAL external APIs
**Status**: ❌ PARTIAL — Apollo real, but LLM fallback generates fake data

**Evidence**:
- `app/apps/web/src/app/api/enrich/route.ts` lines 62-118: Calls Apollo `/v1/organizations/enrich?domain=` — REAL
- Same file lines 120-168: Falls back to `generateObject()` with Claude/GPT — FAKE
- Prompt says "If you don't recognize the company, provide reasonable estimates" — hallucination by design
- Data marked `enrichment_source: "apollo"` (real) vs `"llm_fallback"` (fake)
- `inngest/functions.ts` lines 25-100: Background enrichment on `company/created` — **100% LLM, no Apollo call**

**Effort**: Replace LLM fallback with "data unavailable" state. Fix Inngest to call Apollo API. ~2h

### 1.4 ML scoring is based on REAL signals from external sources
**Status**: ❌ MIXED

**Evidence**:
- `app/api/score/route.ts`: Company scoring uses `calculateFitScore()` (rule-based) + `calculateEngagementScore()` (real activities) — OK
- `app/api/score-contacts/route.ts`: Contact scoring is **100% LLM-based** (lines 22-82), no external API
- Scores generated by Claude/GPT `generateObject()` with 0-100 range — arbitrary
- Scores depend on company data which may itself be LLM-fabricated

**Effort**: Replace LLM contact scoring with rule-based scoring using real signals. ~2h
