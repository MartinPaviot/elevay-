## Pires défauts

1. Voice section refresh() swallows fetch errors with empty catch (page.tsx:467-469), so a /api/calls/config failure silently renders the 'not configured' state with no error toast/banner — a minor H2-flavored degradation on an otherwise H1 page
2. Voice 'configured' status is derived from env (isVoiceConfigured(), calls/config/route.ts:13,18) not per-tenant persisted data — correct for an env capability flag but not tenant-scoped; the pool/usage it gates ARE tenant-scoped
3. Parent page is client-rendered and not itself admin-gated (only PUT/PATCH enforce requireAdmin); GET caps/mailboxes read paths rely on route-level auth, so a non-admin could load (but not mutate) the surface — acceptable but worth noting for the admin-only intent stated in mailboxes/route.ts:8
