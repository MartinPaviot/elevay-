# 16 — meetings (`/meetings`) — audit d'hydratation

**Verdict global : H2 (partiel).** The /meetings page is largely faithful (H1): every data-bearing element — calendar grid, list cards, next-meeting countdown, conflict banner, show-rate chip, CRM account/contact matching, notes/transcript badges, and attendance controls — is wired to real tenant-scoped data from GET /api/meetings, which fans out to Google/Microsoft/CalDAV, joins activities, and matches attendees to tenant contacts/companies (all eq(tenantId)). It has a proper loading skeleton and two distinct written empty states. The single meaningful defect is error handling: the client fetch swallows non-ok responses and the route's own 500, falling through to the 'No meetings in view' empty state, so a backend error is silently presented as 'no meetings' — below the Home reference bar's independent error degradation. The notetaker empty-state copy is gated on a deployment-wide env flag (recallai) rather than tenant data, which is acceptable by design but not tenant-scoped.

Entrée : `app/apps/web/src/app/(dashboard)/meetings/page.tsx`.

## Pires défauts

1. List-fetch error is silently masked as empty: on !res.ok or a thrown fetch (the route can return 500 at app/apps/web/src/app/api/meetings/route.ts:355), page.tsx:79-89 only logs to console and sets loading=false, so the user sees the 'No meetings in view' empty state instead of an error — no independent error degradation (app/apps/web/src/app/(dashboard)/meetings/page.tsx:75-91).
2. Notetaker capability copy depends on a deployment-wide env flag, not tenant data: d.recallai comes from RECALL_API_KEY (app/apps/web/src/app/api/features/route.ts:20) and toggles the empty-state promise at page.tsx:222-244 — not tenant-scoped, so it cannot reflect a per-tenant notetaker configuration.
3. No partial-failure surface: when one calendar provider fails, route.ts uses allSettled and silently returns the others (route.ts:83-100) with no flag to the UI, so the user is never told a feed is degraded/stale (app/apps/web/src/app/api/meetings/route.ts:65-100).
