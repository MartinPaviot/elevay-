# S15 — settings-llm-evals (`/settings/llm-evals`) — audit d'hydratation

**Verdict global : H1 (fidèle).** This is an admin-gated global LLM observability dashboard, not a conventional tenant settings page with persisted toggles. Every data-bearing element is wired to real DB tables (llm_calls, llm_eval_runs, llm_eval_case_runs) via /api/admin/llm-evals and /api/admin/eval-runs/[id]/cases, with first-class loading (AIThinking/spinner), written empty states, and error handling (role=alert, explicit 403 message). It is intentionally cross-tenant: admin-only via requireAdmin, llm_calls.tenantId is nullable for system/eval calls and is not filtered, and the eval-run tables have no tenant column by design, so tenantScoped is n/a not a leak. No mock data, no unwired panels; faithful data hydration for an admin surface.
