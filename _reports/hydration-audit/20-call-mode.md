# 20 — call-mode (`/call-mode`) — audit d'hydratation

**Verdict global : H2 (partiel).** Call Mode is a large, genuinely data-wired cockpit: queue, brief, prospect brief, account brain, funnel KPIs, caller-ID pool, debrief and SSE call events all trace to real tenant-scoped sources (every route uses withAuthRLS / getAuthContext with eq(tenantId)), with mostly faithful loading/empty/error degradation and honest written empties matching the Home-page bar. The fidelity gap is in the DEFAULT (campaign) queue path: campaign/route.ts hardcodes accessibilityScore=0.7 and emits empty localTime/localTimezone, so the per-row reachability pill, the ReachabilitySummary, and the accessibility-driven gap line run off a constant rather than real data (the genuine values exist only in the ?accounts= buildQueue path). The live transcript + execution levers are correctly plumbed to SSE but, per the code's own header, depend on a Phase-1.5 streaming bridge that commonly isn't feeding chunks yet, so they sit empty with an honest diagnostic message. Worst meaningful state is H2 (partial/placeholder data on otherwise-faithful elements); no H4/H5 — no unwired skeletons and no tenant leaks found.

Entrée : `app/apps/web/src/app/(dashboard)/call-mode/page.tsx`.

## Pires défauts

1. Campaign-mode queue hardcodes accessibilityScore: 0.7 for every contact (app/apps/web/src/app/api/calls/campaign/route.ts:101) — the per-row reachability pill (page.tsx:1659), ReachabilitySummary (page.tsx:1592) and the 'Numéro non qualifié (standard probable)' gap line (_panels.tsx:594) all read a constant, not real reachability. Real values exist only in the buildQueue path (lib/voice/queue.ts:142).
2. Campaign-mode queue emits localTime:'' and localTimezone:'' (app/apps/web/src/app/api/calls/campaign/route.ts:106-107), so the local-time clock chip (page.tsx:1671) and the 'heure locale' ContextChip (_panels.tsx:690) never render on the default path — data absent rather than computed (queue.ts:167 computes it for the ?accounts= path).
3. Live transcript + post-call execution levers (page.tsx:1824, _panels.tsx:1211/1297) are wired to the /api/calls/[id]/events SSE but depend on the Phase-1.5 Media-Streams bridge that the file header (page.tsx:8) and the empty-state copy (_panels.tsx:1246) admit isn't shipping chunks today — commonly empty in practice.
