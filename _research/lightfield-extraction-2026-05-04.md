## Competitive Analysis: Lightfield vs Elevay

### Where Lightfield is ahead:
1. **Skills system** — Composable, teachable agent workflows. Users define via NL, not code
2. **Knowledge Layer** — Structured context that Skills reference. Separate from prompts
3. **Custom Objects** — Flexible data model (Pro tier), agent-suggested schema changes
4. **Model-agnostic** — Claude 4.7, GPT 5.5, Gemini 3 Pro. User picks
5. **Workflow Engine** — Durable execution, MCP connectors, agent steps
6. **REST API maturity** — 44 endpoints, 4 SDKs, CLI, OpenAPI spec
7. **MCP Server** — External agents can use Lightfield as a tool
8. **HIPAA compliance** — Healthcare vertical ready
9. **Background tasks** — Long-running agent ops with progress + resume

### Where Lightfield is behind/weak:
1. **No DELETE API** — Cannot programmatically delete anything
2. **No bulk operations** — Max 25 records/page, no batch CRUD
3. **No webhook API** — Webhooks only via workflow UI
4. **No activity/audit log API** — Can't read the activity feed programmatically
5. **No search API** — Only basic field filtering, no full-text search
6. **Auto-enrichment accuracy ~20-40%** — Our testing showed frequent wrong entity matches
7. **Note content recall failure** — Agent found note titles but failed to read content (Query 21)
8. **25 record page cap** — Painful for data sync at scale
9. **Growth tier gates core features** — Account scoring, sequences behind custom pricing

### Architecture decisions to learn from:
1. POST for updates (not PATCH) — idempotent by design
2. `$`-prefixed system fields vs bare custom fields — clear namespace
3. Stainless SDK generation — auto-generates SDKs from OpenAPI
4. Definitions endpoints — schema introspection per entity type
5. File upload flow: create → upload → complete (3-step)
6. Knowledge files with purpose tags (`knowledge_user`, `knowledge_workspace`)
7. Privacy-aware meetings with access levels and per-caller redaction
